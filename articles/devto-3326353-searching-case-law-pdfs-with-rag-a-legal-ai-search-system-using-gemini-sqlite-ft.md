## Challenges in Case Law Search

Traditional court databases (e.g., courts.go.jp) have search functions limited to string matching, and there is no way to search the text within PDFs. Since PDFs contained in case law collections are not extracted as text data, it is common for relevant cases not to be found even when entering keywords like "民法90条" (Civil Code Article 90). Furthermore, due to the lack of technology to automatically extract "争点" (issues) and "判示事項" (points of law), which are central to a judgment, legal professionals had to manually interpret the details of case law.

This system resolves these challenges by converting PDFs into readable text, enabling high-speed search with SQLite's FTS5 (Full-Text Search), and automating case law analysis using Gemini.

## System Architecture

This system consists of a four-stage pipeline.

### PDF Parsing
PDFs are converted to text using PyPDF2. For scanned case law, OCR (Tesseract) is used in conjunction, and an index is built in the SQLite database. BM25 weighting is applied to FTS5, optimized for the specific grammar of Japanese case law.

### SQLite FTS5
A search index is created with the following SQL to accelerate keyword searches.

```sql
CREATE VIRTUAL TABLE hanrei_search USING fts5(id, text);
INSERT INTO hanrei_search VALUES(1, '民法第90条に基づく...');
```

### Gemini Analysis
The case law text is analyzed using Gemini 2.5 Pro. The prompt for issue extraction is "この判例の争点を3項目以内にまとめよ" (Summarize the issues of this case in 3 items or less), and for judgment summary, it instructs "判示事項を100字で要約せよ" (Summarize the points of law in 100 characters).

### Streamlit UI
Gemini analysis results are displayed instantly with the search results. FTS5's BM25 score is utilized for recommending similar cases, calculating practical similarity.

## Case Law Analysis with Gemini API

Gemini 2.5 Pro can accurately extract the core of case law. An example implementation code is shown below.

```python
from google import genai

client = genai.Client()

prompt = "この判例の争点を3項目以内にまとめよ。判示事項を100字で要約せよ。\n\n" + pdf_text
response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=prompt
)
print(response.text)
```

## Implementing OTP Email Authentication

Preventing unauthorized access is essential when handling legal data. This system implemented email OTP authentication with Firebase Authentication.
- A 6-digit OTP is sent via email upon login (valid for 5 minutes).
- The account is locked after 3 failed attempts (rate limiting).
- Communication is encrypted with TLS 1.3, and data is stored in encrypted storage.
- For GDPR compliance, data is stored only on the user's device.

## Applying Patent Search Technology to Case Law

Proven technologies from patent search systems have been applied to the legal domain.
- FTS5 Japanese Optimization: BM25 parameters (k1, b) were adjusted to match the grammar of case law. "民法" (Civil Code) and "民法第90条" (Civil Code Article 90) can be treated as related keywords.
- Common Design for Analysis UI: The filter function of patent LLMs was repurposed, allowing case law to be narrowed down by issue type or case year.
- Cloudflare Utilization: DDoS attack countermeasures were introduced, improving response speed.
